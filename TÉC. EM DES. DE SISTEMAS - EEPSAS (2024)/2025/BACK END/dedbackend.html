1º Bimestre (27 aulas)

Semana 1 (06/02, 07/02)

Tema: Introdução ao Desenvolvimento Back-End.
Objetivo: Compreender a arquitetura cliente-servidor e o papel do back-end em aplicações web.
Tópicos:
06/02 (Quinta-feira):
  • Apresentação da disciplina. Arquitetura cliente-servidor.
07/02 (Sexta-feira - 2 aulas):
  • Protocolo HTTP (requests/responses, métodos, status codes).
  • O que é uma API? Diferenças entre REST e SOAP.

Semana 2 (13/02, 14/02)

Tema: Ambiente de Desenvolvimento Node.js.
Objetivo: Configurar o ambiente de desenvolvimento e entender o ecossistema Node.js.
Tópicos:
13/02 (Quinta-feira):
  • Instalação do Node.js e NPM. O que é o NPM?
14/02 (Sexta-feira - 2 aulas):
  • Iniciando um projeto (`npm init`). Estrutura do `package.json`.
  • Módulos CommonJS (`require`/`module.exports`).

Semana 3 (20/02, 21/02)

Tema: Introdução ao Framework Express.js.
Objetivo: Aprender a criar um servidor web e definir rotas básicas com Express.
Tópicos:
20/02 (Quinta-feira):
  • Instalando o Express. Criando o primeiro servidor "Hello World".
21/02 (Sexta-feira - 2 aulas):
  • Definindo rotas com métodos HTTP (`app.get`, `app.post`).
  • Enviando respostas para o cliente (`res.send`, `res.json`).

Semana 4 (27/02, 28/02)

Tema: Rotas, Requisições e Respostas.
Objetivo: Aprofundar o manuseio de requisições, capturando parâmetros e dados.
Tópicos:
27/02 (Quinta-feira):
  • Capturando Route Params (`req.params`) e Query Strings (`req.query`).
28/02 (Sexta-feira - 2 aulas):
  • Middleware `express.json()`.
  • Capturando dados do corpo da requisição (`req.body`) em rotas POST/PUT.

Semana 5 (06/03, 07/03)

Tema: Padrão REST e Construção de APIs.
Objetivo: Entender os princípios REST para a construção de APIs consistentes.
Tópicos:
06/03 (Quinta-feira):
  • Princípios REST (Recursos, Verbos HTTP, Stateless).
07/03 (Sexta-feira - 2 aulas):
  • Testando APIs com Postman/Insomnia.
  • Atividade: Estruturando os endpoints de uma API RESTful para um recurso (ex: Produtos).

Semana 6 (13/03, 14/03)

Tema: Middlewares em Express.
Objetivo: Compreender o conceito de middlewares para modularizar o tratamento de requisições.
Tópicos:
13/03 (Quinta-feira):
  • O que são middlewares? A função `next()`.
14/03 (Sexta-feira - 2 aulas):
  • Criando middlewares globais e específicos para rotas.
  • Exemplo prático: criando um middleware de log de requisições.

Semana 7 (20/03, 21/03)

Tema: Organização e Estrutura de Projetos.
Objetivo: Aprender a organizar o código de um projeto back-end de forma escalável.
Tópicos:
20/03 (Quinta-feira):
  • Padrão MVC (Model-View-Controller) adaptado para APIs.
21/03 (Sexta-feira - 2 aulas):
  • Refatorando o projeto: separando rotas (`routes.js`).
  • Separando controladores (`controllers.js`) da lógica das rotas.

Semana 8 (27/03, 29/03 - Sábado Letivo)

Tema: Introdução à Persistência de Dados.
Objetivo: Conectar a aplicação Node.js a um banco de dados relacional.
Tópicos:
27/03 (Quinta-feira):
  • Revisão de conceitos de Banco de Dados e SQL.
29/03 (Sábado - Horário de Quinta):
  • Conectando ao MySQL/PostgreSQL com um driver (`mysql2`/`pg`).

Semana 9 (03/04, 04/04)

Tema: CRUD com Banco de Dados.
Objetivo: Implementar as operações básicas (Create, Read, Update, Delete) em uma API.
Tópicos:
03/04 (Quinta-feira):
  • Implementando as rotas de Leitura (Read - `SELECT`).
04/04 (Sexta-feira - 2 aulas):
  • Implementando a rota de Criação (`INSERT`).
  • Implementando as rotas de Atualização e Deleção (`UPDATE`, `DELETE`).

Semana 10 (10/04, 11/04)

Tema: Revisão e Atividades Avaliativas - 1º Bimestre.
Objetivo: Consolidar os conhecimentos na criação de uma API RESTful com persistência de dados.
Tópicos:
10/04 (Quinta-feira):
  • Revisão geral dos tópicos do bimestre. Tira-dúvidas.
11/04 (Sexta-feira - 2 aulas):
  • Avaliação prática: desenvolver uma API CRUD completa para um recurso.

2º Bimestre (22 aulas)

Semana 11 (02/05)

Tema: JavaScript Assíncrono.
Objetivo: Compreender a natureza assíncrona do Node.js e aprender a lidar com operações de I/O.
Tópicos:
02/05 (Sexta-feira - 2 aulas):
  • Callbacks e o problema do "Callback Hell".
  • Introdução a Promises como solução.

Semana 12 (08/05, 09/05)

Tema: Promises e Async/Await.
Objetivo: Utilizar Promises e a sintaxe Async/Await para escrever código assíncrono limpo.
Tópicos:
08/05 (Quinta-feira):
  • Promises: `.then()`, `.catch()`, `.finally()`.
09/05 (Sexta-feira - 2 aulas):
  • Sintaxe `async/await` para simplificar o código.
  • Tratamento de erros com `try/catch` em blocos `async`.

Semana 13 (15/05, 16/05)

Tema: Prevenção de SQL Injection.
Objetivo: Entender a vulnerabilidade de SQL Injection e preveni-la com consultas parametrizadas.
Tópicos:
15/05 (Quinta-feira):
  • O que é SQL Injection? Exemplos práticos de ataques.
16/05 (Sexta-feira - 2 aulas):
  • Consultas parametrizadas (Prepared Statements).
  • Atividade: Refatorando o CRUD para torná-lo seguro contra injeção de SQL.

Semana 14 (22/05, 23/05)

Tema: Variáveis de Ambiente.
Objetivo: Aprender a gerenciar configurações e segredos da aplicação de forma segura.
Tópicos:
22/05 (Quinta-feira):
  • Problema de "hardcoding" de senhas e configurações no código.
23/05 (Sexta-feira - 2 aulas):
  • Uso de arquivos `.env` para armazenar variáveis de ambiente.
  • Implementação com a biblioteca `dotenv`.

Semana 15 (05/06, 06/06) - Prova Parcial

Tema: Avaliação Parcial.
Objetivo: Avaliar a aplicação de código assíncrono e seguro na construção de APIs.
Tópicos:
05/06 (Quinta-feira):
  • Aplicação de avaliação prática no laboratório.
06/06 (Sexta-feira - 2 aulas):
  • Continuação da avaliação, correção e feedback.

Semana 16 (12/06, 13/06)

Tema: Autenticação e Sessões.
Objetivo: Compreender os mecanismos de autenticação e gerenciamento de estado.
Tópicos:
12/06 (Quinta-feira):
  • Autenticação vs. Autorização.
13/06 (Sexta-feira - 2 aulas):
  • Gerenciamento de estado com Sessões e Cookies.
  • Implementando um sistema de login básico com a biblioteca `express-session`.

Semana 17 (26/06, 28/06 - Sábado Letivo) - Prova Global

Tema: Avaliação Global e Revisão Final.
Objetivo: Avaliar o conhecimento do bimestre e revisar para recuperação.
Tópicos:
26/06 (Quinta-feira):
  • Aplicação da Prova Global do 2º Bimestre.
28/06 (Sábado - Horário de Sexta - 2 aulas):
  • Correção e feedback da prova.
  • Preparação para a semana de recuperação.

Semana 18 (03/07, 04/07) - Recuperação

Tema: Semana de Recuperação Parcial.
Objetivo: Oferecer nova oportunidade de aprendizado e avaliação.
Tópicos:
03/07 (Quinta-feira):
  • Aula de reforço com foco em JavaScript assíncrono e CRUD seguro.
04/07 (Sexta-feira - 2 aulas):
  • Aplicação de atividades/prova de recuperação. Atendimento individualizado.

3º Bimestre (23 aulas)

Semana 19 (07/08, 08/08)

Tema: Criptografia de Senhas.
Objetivo: Aprender a armazenar senhas de usuários de forma segura no banco de dados.
Tópicos:
07/08 (Quinta-feira):
  • Por que não se deve armazenar senhas em texto puro? Hashing vs. Criptografia.
08/08 (Sexta-feira - 2 aulas):
  • Introdução à biblioteca `bcrypt`.
  • Atividade: gerar hashes de senhas e comparar senhas no login.

Semana 20 (14/08)

Tema: Autenticação Baseada em Token (JWT).
Objetivo: Implementar um sistema de autenticação stateless utilizando JSON Web Tokens.
Tópicos:
14/08 (Quinta-feira):
  • O que é JWT? Estrutura de um token (Header, Payload, Signature).

Semana 21 (21/08, 22/08, 23/08 - Sábado Letivo)

Tema: Implementação de JWT.
Objetivo: Gerar, enviar e validar tokens JWT em uma API.
Tópicos:
21/08 (Quinta-feira):
  • Uso da biblioteca `jsonwebtoken` para criar e assinar tokens.
22/08 (Sexta-feira - 2 aulas):
  • Verificação do token em requisições.
23/08 (Sábado - Horário de Quinta):
  • Criando um middleware para proteger rotas.

Semana 22 (28/08, 29/08)

Tema: Upload de Arquivos.
Objetivo: Aprender a receber e processar arquivos enviados pelo cliente.
Tópicos:
28/08 (Quinta-feira):
  • Lidando com requisições `multipart/form-data`.
29/08 (Sexta-feira - 2 aulas):
  • Introdução à biblioteca `multer` para gerenciar o upload de arquivos.
  • Atividade prática: upload de imagem de perfil de usuário.

Semana 23 (04/09, 05/09)

Tema: APIs Web e Comunicação Externa.
Objetivo: Compreender como o back-end pode consumir serviços de outras APIs.
Tópicos:
04/09 (Quinta-feira):
  • Padrões de APIs externas. Foco em REST.
05/09 (Sexta-feira - 2 aulas):
  • Consumindo APIs de terceiros com `axios` ou `node-fetch`.
  • Exemplo: buscar dados de uma API pública (ex: CEP, clima).

Semana 24 (11/09, 13/09 - Sábado Letivo)

Tema: Revisão e Atividades Avaliativas.
Objetivo: Consolidar o aprendizado sobre autenticação, segurança e comunicação.
Tópicos:
11/09 (Quinta-feira):
  • Revisão geral dos tópicos do bimestre. Tira-dúvidas.
13/09 (Sábado - Horário de Sexta - 2 aulas):
  • Prova prática: implementar autenticação JWT em uma API CRUD.

4º Bimestre (24 aulas)

Semana 25 (23/10, 24/10)

Tema: Introdução a Testes de Software.
Objetivo: Compreender a importância dos testes e os diferentes tipos.
Tópicos:
23/10 (Quinta-feira):
  • Por que testar? Pirâmide de testes (unitário, integração, E2E).
24/10 (Sexta-feira - 2 aulas):
  • Introdução ao framework de testes Jest.
  • Configurando o Jest em um projeto Node.js.

Semana 26 (30/10, 31/10)

Tema: Testes Unitários.
Objetivo: Aprender a escrever testes para pequenas unidades de código (funções).
Tópicos:
30/10 (Quinta-feira):
  • Escrevendo o primeiro teste com `describe`, `it`, `expect`.
31/10 (Sexta-feira - 2 aulas):
  • Matchers do Jest (`toBe`, `toEqual`, `toBeTruthy`, etc.).
  • Testando funções puras do projeto.

Semana 27 (06/11, 07/11)

Tema: Testes de Integração para APIs.
Objetivo: Aprender a testar os endpoints da API, simulando requisições HTTP.
Tópicos:
06/11 (Quinta-feira):
  • Configurando o ambiente para testes de API com `supertest`.
07/11 (Sexta-feira - 2 aulas):
  • Escrevendo testes para rotas GET e POST.
  • Escrevendo testes para rotas PUT e DELETE, validando status codes e retornos.

Semana 28 (13/11, 14/11)

Tema: Implantação (Deploy) de Aplicações.
Objetivo: Entender os conceitos básicos para publicar uma aplicação na internet.
Tópicos:
13/11 (Quinta-feira):
  • O que é deploy? Conceitos de PaaS (Platform as a Service).
14/11 (Sexta-feira - 2 aulas):
  • Preparando a aplicação para o deploy (scripts, porta).
  • Demonstração de deploy em uma plataforma como Railway, Render ou Vercel.

Semana 29 (21/11)

Tema: Definição do Projeto Final.
Objetivo: Estruturar o projeto final que integrará todos os conceitos do ano.
Tópicos:
21/11 (Sexta-feira - 2 aulas):
  • Apresentação da proposta do projeto.
  • Formação dos grupos e brainstorming inicial da arquitetura.

Semana 30 (27/11, 28/11)

Tema: Desenvolvimento do Projeto Final.
Objetivo: Desenvolver o projeto final com a mentoria do professor.
Tópicos:
27/11 (Quinta-feira):
  • Desenvolvimento assistido do projeto no laboratório.
28/11 (Sexta-feira - 2 aulas):
  • Tira-dúvidas e ajustes na arquitetura e implementação dos projetos.

Semana 31 (04/12, 06/12 - Sábado Letivo)

Tema: Finalização e Apresentação do Projeto.
Objetivo: Concluir e apresentar o projeto, demonstrando o domínio das tecnologias.
Tópicos:
04/12 (Quinta-feira):
  • Últimos ajustes e preparação para a apresentação final.
06/12 (Sábado - Horário de Sexta - 2 aulas):
  • Apresentação dos projetos finais para a turma.
  • Avaliação do projeto e encerramento da disciplina.
